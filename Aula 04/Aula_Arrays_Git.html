🎯 O que são Arrays?

🏠 Analogia: Prédio de Apartamentos

Imagine um prédio com vários apartamentos numerados. Cada apartamento pode guardar um item do mesmo tipo.

📋 Definição

Arrays são coleções de elementos do mesmo tipo, organizados em posições sequenciais (índices).

✨ Características

  • Tamanho fixo - definido na criação
  • Mesmo tipo - todos elementos iguais
  • Índice numérico - começa em 0
  • Acesso direto - O(1) por posição

📋 ArrayList - Listas Dinâmicas

🎒 Analogia: Mochila Mágica

Imagine uma mochila que cresce automaticamente quando você adiciona mais itens. Não precisa definir o tamanho antes!

🆚 Array vs ArrayList

🔒 Array Tradicional:

  • Tamanho fixo - definido na criação
  • Mais rápido - acesso direto
  • Menos memória - sem overhead
  • Primitivos - int, double, etc.

🚀 ArrayList:

  • Tamanho dinâmico - cresce automaticamente
  • Métodos úteis - add, remove, contains
  • Flexível - adiciona/remove facilmente
  • Objetos - String, Integer, etc.
import java.util.ArrayList; // Array tradicional - FIXO int[] numeros = new int[5]; // Só 5 elementos! // ArrayList - DINÂMICO ArrayList<Integer> lista = new ArrayList<>(); lista.add(10); // Adiciona lista.add(20); // Cresce automaticamente lista.add(30); // Sem limite! // Tamanho atual System.out.println("Tamanho: " + lista.size());

🎯 Array vs ArrayList - Quando Usar?

🔒 Use Arrays Quando:

  • Tamanho conhecido - Ex: dias da semana (7)
  • Performance crítica - jogos, cálculos pesados
  • Tipos primitivos - int, double, boolean
  • Matrizes - int[][] tabuleiro
  • Memória limitada - sistemas embarcados
// Exemplos ideais para Arrays: int[] diasMes = {31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31}; double[] coordenadas = {10.5, 20.3}; // x, y boolean[] respostas = new boolean[10]; // Quiz de 10 questões

🚀 Use ArrayList Quando:

  • Tamanho variável - lista de usuários
  • Adicionar/remover - carrinho de compras
  • Busca frequente - contains(), indexOf()
  • Facilidade de uso - métodos prontos
  • Objetos complexos - String, custom classes
// Exemplos ideais para ArrayList: ArrayList<String> carrinho = new ArrayList<>(); // Cresce conforme compra ArrayList<Usuario> usuarios = new ArrayList<>(); // Lista de cadastros ArrayList<String> historico = new ArrayList<>(); // Histórico de navegação

🏠 Analogia Prática

🔒 Array = Apartamento

Tamanho fixo, mais barato, ideal quando você sabe exatamente quantos quartos precisa.

🚀 ArrayList = Casa Expansível

Pode adicionar quartos, mais caro, ideal quando a família pode crescer.

🔄 O que é Git?

📚 Analogia: Máquina do Tempo para Código

Imagine ter uma máquina do tempo que salva diferentes versões do seu projeto e permite voltar a qualquer momento!

🎯 Definição

Git é um sistema de controle de versão distribuído que rastreia mudanças em arquivos e coordena trabalho entre múltiplas pessoas.

🚀 Por que usar Git?

  • Histórico completo - nunca perca código
  • Trabalho em equipe - colaboração eficiente
  • Branches - desenvolva recursos isoladamente
  • Backup automático - código sempre seguro
  • Rastreabilidade - quem mudou o quê e quando

🌍 Git no Mundo Real

  • Linux: Criado por Linus Torvalds
  • GitHub: +100 milhões de repositórios
  • Empresas: Google, Microsoft, Facebook
  • Open Source: 99% dos projetos usam Git

📈

Git é o padrão da indústria!

🧠 Conceitos Fundamentais do Git

📂 Repository (Repositório)

Uma pasta especial que o Git monitora. Contém todo o histórico do projeto.

git init meu-projeto

📸 Commit

Um "snapshot" do seu código em um momento específico. Como uma foto do projeto.

git commit -m "Adiciona login de usuário"

🌿 Branch

Uma linha paralela de desenvolvimento. Permite trabalhar em recursos sem afetar o código principal.

git branch nova-feature

🔄 Merge

Combinar mudanças de diferentes branches. Junta o trabalho de várias pessoas.

git merge nova-feature

🎬 Analogia: Produção de Filme

Repository = Estúdio | Commit = Cena gravada | Branch = Roteiro alternativo | Merge = Edição final

✨ Boas Práticas Git

📝 Mensagens de Commit

// ❌ Ruim git commit -m "mudança" git commit -m "fix" // ✅ Bom git commit -m "feat: adiciona sistema de login" git commit -m "fix: corrige bug no cálculo de preços" git commit -m "docs: atualiza README com instruções"

Padrão: tipo(escopo): descrição

  • feat: nova funcionalidade
  • fix: correção de bug
  • docs: documentação
  • style: formatação
  • refactor: refatoração

🌿 Estratégia de Branches

main/master ← Código em produção develop ← Desenvolvimento principal feature/login ← Nova funcionalidade hotfix/bug ← Correção urgente

Git Flow:

  • main: código estável
  • develop: integração contínua
  • feature/*: novas funcionalidades
  • hotfix/*: correções urgentes

⚠️ O que NÃO fazer

  • Committar diretamente na main
  • Commits gigantes com muitas mudanças
  • Mensagens vagas: "fix", "update"
  • Commitar arquivos temporários
  • Ignorar conflitos de merge

🔧 Comandos Git Avançados

🕐 Histórico e Logs

git log --oneline # Log resumido git log --graph # Visualização gráfica git log --author="João" # Commits por autor git show [commit-hash] # Detalhes do commit git diff # Mudanças não commitadas git diff --staged # Mudanças em staging

💾 Stash (Guardar Temporário)

git stash # Guarda mudanças git stash list # Lista stashes git stash pop # Aplica último stash git stash apply stash@{0} # Aplica stash específico git stash drop # Remove stash

⏪ Reset e Revert

git reset --soft HEAD~1 # Desfaz commit (mantém staging) git reset --hard HEAD~1 # Desfaz commit (perde mudanças) git revert [commit-hash] # Cria commit que desfaz outro git checkout [commit-hash] # Volta para commit específico

⚠️ Comandos Perigosos

  • git reset --hard: Perde mudanças permanentemente
  • git push --force: Reescreve histórico remoto
  • git clean -fd: Remove arquivos não rastreados

💡 Dica: Sempre faça backup antes de usar comandos destrutivos!

⚔️ Resolvendo Conflitos

🤝 Analogia: Negociação

Conflitos são como duas pessoas editando o mesmo documento. O Git pede para você escolher qual versão manter.

❗ Como Conflitos Acontecem

  • Duas pessoas editam a mesma linha
  • Merge de branches divergentes
  • Pull de mudanças conflitantes
  • Rebase com mudanças sobrepostas

🔍 Identificando Conflitos:

git status # Mostra arquivos em conflito git diff # Visualiza diferenças

🛠️ Marcadores de Conflito

<<<<<<< HEAD String nome = "João"; ======= String nome = "Maria"; >>>>>>> feature/novo-nome

Significado:

  • <<<<<<< HEAD: Sua versão atual
  • =======: Separador
  • >>>>>>> branch: Versão do outro branch

🔧 Processo de Resolução

1. git pull origin main # Identifica conflito 2. [Editar arquivos manualmente] # Remove marcadores, escolhe versão 3. git add arquivo.java # Marca como resolvido 4. git commit # Finaliza merge 5. git push origin main # Envia resolução

💡 Dicas para Evitar Conflitos

  • Pull frequente: git pull antes de trabalhar
  • Commits pequenos: Mudanças menores = menos conflitos
  • Comunicação: Coordene com a equipe
  • Branches curtos: Merge rápido de features
